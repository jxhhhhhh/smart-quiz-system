## 二、系统逻辑合理性评价

### 2.1 架构分层

系统采用四层架构：

```
用户交互层（web_interface.py + page_modules/）
    ↓
业务逻辑层（question_importer.py + practice_session.py + ai_enhancement.py）
    ↓
数据处理层（parser.py + ai_parser.py）
    ↓
数据存储层（database.py）
```

**评价**：分层合理，各层职责清晰。`web_interface.py` 仅负责路由分发（94行），具体页面渲染委托给 `page_modules/` 下的独立模块，实现了高内聚低耦合。`parser.py` 提供统一的 `check_answer()` 函数供上层调用，消除了重复判断逻辑。所有页面模块已统一使用 `db.execute()` 访问数据库，不再直接操作 cursor。

### 2.2 模块耦合与内聚

| 模块 | 内聚度 | 耦合度 | 评价 |
|------|--------|--------|------|
| database.py | 高 | 低 | 单例模式，所有模块通过统一接口访问 |
| parser.py | 高 | 低 | 纯函数，无外部依赖，可独立测试 |
| ai_parser.py | 高 | 低 | 依赖 openai 库，与 database 无直接耦合 |
| question_importer.py | 中 | 中 | 依赖 parser + database + ai_parser |
| practice_session.py | 中 | 中 | 依赖 database + parser |
| page_modules/ | 中 | 低 | 统一使用 `db.execute()`，遵循分层原则 |

**评价**：模块耦合度整体较低，分层原则执行到位。`page_modules/` 已不再直接操作数据库 cursor，所有数据访问均通过 `db.execute()` 封装方法完成。

### 2.3 数据流闭环性

```
导入 → 存储 → 练习 → 记录 → 统计 → 错题 → 重练
  ↑                                           ↓
  └───────────────────────────────────────────┘
```

**评价**：数据流完整闭环。导入的题目进入 `questions` 表，练习产生的记录进入 `practice_records` 表，统计模块从 `practice_records` 聚合数据，错题模块关联两表生成错题列表，错题重练回到练习环节。每个环节均有对应的数据库操作和 UI 展示。

**验证**：
- 导入 → 存储：`question_importer._insert_questions()` ✅
- 存储 → 练习：`PracticeSession.get_questions()` ✅
- 练习 → 记录：`practice.py` 中通过 `db.execute()` 写入 ✅
- 记录 → 统计：`statistics.py` 中的 SQL 聚合 ✅
- 统计 → 错题：`wrong_review.py` 中的 JOIN 查询 ✅
- 错题 → 重练：`wrong_review.py` 中的页面跳转 ✅
