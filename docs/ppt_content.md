## 第2页：01 我发现了什么问题？

### 真实场景

大学生在备考《Python程序设计》等课程时，需要大量刷题巩固知识点。

### 谁遇到了问题？

- 大学生（尤其是编程类课程学习者）
- 从学习通、超星等平台复制的题目无法高效整理和复习

### 问题表现

| 问题 | 具体表现 |
|------|---------|
| 📄 题目管理混乱 | 从学习通复制的题目格式杂乱，包含"教师批阅""得X分"等干扰信息，手动整理耗时耗力 |
| ❌ 错题无法自动汇总 | 做错的题目散落在各次练习中，无法集中复习薄弱环节 |
| 📊 学习效果难以量化 | 不知道自己哪些题型弱、正确率如何变化，复习缺乏针对性 |
| 🔄 复习策略不科学 | 没有依据遗忘曲线安排复习，考前突击效果差、知识遗忘快 |

---

## 第3页：02 为什么值得解决？

### 现状调查

- 大学生平均每周花费 **3-5小时** 进行课程复习刷题
- 其中约 **30%** 的时间浪费在题目整理和格式清洗上
- 错题复习是提升成绩最有效的方式，但 **70%** 的学生没有系统的错题管理习惯
- 艾宾浩斯遗忘曲线表明，学习后第1、2、4、7、15、30天是最佳复习时间点，但人工追踪几乎不可能

### 问题影响

- **时间浪费**：手动整理题目格式，重复劳动
- **复习低效**：没有针对性，不知道薄弱点在哪
- **记忆衰减**：缺乏科学的间隔复习策略，考前突击遗忘快

**结论：** 一个能够自动解析题目、智能管理错题、科学安排复习、量化学习效果的工具，具有明确的实际需求。

## 第5页：04 核心功能 — 智能题目导入

### 解决：题目管理混乱问题

- 支持**四种来源**：文本粘贴、PDF文件、Word文档、图片OCR
- **双层解析引擎**（核心亮点）：
  - 正则高速解析：20+预编译正则，1000题/秒
  - AI智能兜底：接入 DeepSeek/MIMO API，对乱序文本智能识别
  - **置信度路由**：≥0.8直接采纳 → 0.5~0.8 AI修复 → <0.5 AI重解析
- 自动清洗格式（全角转半角、零宽字符清除）
- **智能去除干扰信息**：自动过滤"教师批阅""得X分""AI讲解"等噪音
- 语义指纹去重（MD5），避免重复导入
- 质量检查 + 自动修复

```
输入：1.(单选题) 关于视图的描述不正确的是（ ）
      A.视图是从表导出的  B.视图存储了实际数据...
      我的答案：B; 得2分; 教师批阅时间2026-05-10
输出：题型=单选 | 题干=关于视图的描述不正确的是（  ）
      选项={A:视图是从表导出的, B:视图存储了实际数据...}
      答案=B ✅  置信度=0.92 → 直接采纳
```

## 第6页：04 核心功能 — 多模式刷题

### 解决：重复练习效率低问题

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 🎲 随机模式 | 从题库随机抽题 | 日常练习、综合复习 |
| ❌ 错题重练 | 按错误次数排序 | 薄弱知识点强化 |
| ⭐ 难度专项 | 按难度筛选（简单/中等/困难） | 针对性突破 |
| 📅 间隔复习 | 艾宾浩斯遗忘曲线（第1/2/4/7/15/30天） | 科学长期记忆 |

### 核心特性

- 支持按**题型**（单选/多选/判断/填空/简答）和**学科**筛选
- 实时计时，答题后即时反馈对错
- **练习进度持久化**：中途退出可恢复，不丢失进度
- **"上一题/下一题"导航**：答错可回头查看
- **答案判断五层匹配**：精确→去括号→包含→子集→Levenshtein模糊匹配

## 第7页：04 核心功能 — 模拟考试

### 模拟真实考试场景

- **自定义组卷**：设置题目数量、考试时长、题型范围、学科范围
- **限时考试**：实时倒计时 + 进度条
- **低时间警告**：剩余不足5分钟时视觉提醒

### 考试流程

```

    → <5分钟警告 | ≤0自动交卷
    → 逐题评分 → 成绩报告
```

## 第11页：07 关键技术实现

### 核心代码片段

**PBKDF2 密码安全**（100,000次迭代 + 32字符随机盐）：

```python
def _hash_password(password, salt=None):
    iterations = 100_000
    salt = salt or secrets.token_hex(16)
    dk = hashlib.pbkdf2_hmac('sha256', password.encode(),
                              salt.encode(), iterations)
    return f"pbkdf2:{iterations}:{salt}:{dk.hex()}"
```

**统一答案判断**（5层匹配策略）：

```python
def check_answer(user, correct, q_type):
    if q_type in ("fill", "short"):
        if user == correct: return True          # 1.精确
        if clean(user) == clean(correct): return True  # 2.去括号
        if correct in user: return True          # 3.包含
        if parts_match(user, correct): return True     # 4.子集
        if fuzzy_match(user, correct): return True     # 5.模糊
    return normalize(user) == normalize(correct)
```
