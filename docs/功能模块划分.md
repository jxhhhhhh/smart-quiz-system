## 3.2 功能设计

### 功能模块划分

本系统按照业务逻辑将功能划分为六大模块，各模块职责清晰、高内聚低耦合。系统功能模块结构如下：

```
智能刷题系统
├── 题目管理模块
│   ├── 多来源导入（文本/PDF/Word/图片）
│   ├── 双层解析（正则+AI）
│   ├── 质量检查与自动修复
│   ├── 语义去重
│   ├── 题目增删改查
│   └── 收藏与导出
├── 练习模块
│   ├── 随机刷题
│   ├── 错题重练
│   ├── 难度专项
│   ├── 间隔重复复习
│   └── 练习进度持久化
├── 考试模块
│   ├── 自定义组卷
│   ├── 限时考试
│   ├── 自动交卷
│   └── 成绩统计
├── 统计分析模块
│   ├── 学习概况
│   ├── 答题趋势
│   ├── 题型正确率
│   └── 薄弱知识点提示
├── 用户模块
│   ├── 注册登录
│   ├── 密码加密
│   └── 数据隔离
└── 系统模块
    ├── 深色/浅色主题
    ├── 响应式布局
    ├── 日志记录
    └── CSV/JSON导出
```

#### 模块一：题目管理模块

**职责**：负责题目的全生命周期管理，从导入、解析、存储到编辑、收藏和导出。

| 子模块 | 功能说明 |
|--------|---------|
| 多来源导入 | 支持从文本粘贴、PDF 文件、Word 文档、图片（OCR）四种来源批量导入题目，适配不同场景下的题目获取需求 |
| 双层解析 | 采用"正则高速解析 + AI 智能兜底"的双层引擎。正则解析器通过预编译正则表达式实现高速识别（1000题/秒），对置信度不足的结果交给 AI 大语言模型进行修复或重新解析 |
| 质量检查与自动修复 | 自动检测导入题目中的质量问题（空题干、缺失答案、选项异常），对可修复项自动修复，不可修复项标记跳过并生成质量报告 |
| 语义去重 | 基于 MD5 语义指纹（标准化内容 + 排序选项 + 大写答案）实现精确去重，避免重复题目污染题库 |
| 题目增删改查 | 支持题目的新增、编辑（题干/选项/答案/学科）、单题删除、批量删除，提供分页浏览和按题型/学科/难度筛选 |
| 收藏与导出 | 支持一键收藏/取消收藏题目，支持将题库导出为 TXT 或 JSON 格式，方便离线复习或数据迁移 |

#### 模块二：练习模块

**职责**：提供多样化的答题练习模式，支持练习进度的保存与恢复。

| 子模块 | 功能说明 |
|--------|---------|
| 随机刷题 | 从题库中随机抽取指定数量的题目进行练习，适合日常巩固复习 |
| 错题重练 | 按错误次数从高到低排序，优先练习高频错题，针对性强化薄弱环节 |
| 难度专项 | 按难度等级（简单/中等/困难）筛选题目，支持定向突破特定难度 |
| 间隔重复复习 | 基于艾宾浩斯遗忘曲线（第 1、2、4、7、15、30 天），自动推送需要复习的题目，科学安排复习计划 |
| 练习进度持久化 | 每次答题后自动保存进度到数据库，支持中断后恢复练习，避免因意外退出导致进度丢失 |

#### 模块三：考试模块

**职责**：模拟真实考试场景，提供限时考试和自动评分功能。

| 子模块 | 功能说明 |
|--------|---------|
| 自定义组卷 | 用户可设置考试参数，包括题目数量、考试时长、题型范围、学科范围，系统按参数随机抽题生成试卷 |
| 限时考试 | 考试过程中实时显示倒计时和答题进度，剩余不足 5 分钟时发出警告提示 |
| 自动交卷 | 考试时间用尽后系统自动交卷，防止超时答题 |
| 成绩统计 | 考试结束后展示成绩报告，包括总分、正确率、答题用时，并提供逐题的答题详情和解析 |

#### 模块四：统计分析模块

**职责**：对学习数据进行多维度分析，帮助用户量化学习效果、发现薄弱环节。

| 子模块 | 功能说明 |
|--------|---------|
| 学习概况 | 展示近 30 天的核心指标卡片：答题总次数、正确数、正确率，快速了解整体学习状态 |
| 答题趋势 | 使用 Plotly 生成近 7 天每日答题数量柱状图和正确率趋势折线图，直观展示学习进展 |
| 题型正确率 | 按题型（单选/多选/判断/填空/简答）统计正确率，通过柱状图对比各题型的掌握程度 |
| 薄弱知识点提示 | 基于答题数据自动分析薄弱环节，生成智能学习建议（如"多选题正确率较低，建议加强练习"） |

#### 模块五：用户模块

**职责**：管理用户身份认证和数据隔离，保障用户数据安全。

| 子模块 | 功能说明 |
|--------|---------|
| 注册登录 | 支持用户注册和登录，登录后进入个人专属界面，未登录用户可使用基础功能 |
| 密码加密 | 采用 SHA-256 + 随机盐（32 字符）的方式对密码进行哈希存储，防止明文泄露 |
| 数据隔离 | 通过 `user_id` 字段关联用户数据，答题记录、收藏、练习会话等数据按用户隔离，互不影响 |

#### 模块六：系统模块

**职责**：提供全局性的系统级功能，提升用户体验和系统可维护性。

| 子模块 | 功能说明 |
|--------|---------|
| 深色/浅色主题 | 支持深色和浅色两套主题切换，通过自定义 CSS（773 行）实现，适配不同使用环境 |
| 响应式布局 | 针对移动端设备添加媒体查询，在小屏幕设备上自动调整布局，保障移动端可用性 |
| 日志记录 | 使用统一的日志模块记录系统运行日志，便于问题排查和运行监控 |
| CSV/JSON导出 | 支持将答题历史、题库数据导出为 CSV（UTF-8 BOM，兼容 Excel）或 JSON 格式，方便数据分析和备份 |

### 模块功能描述

#### 模块名称：题目导入与解析模块

**描述**：用户可通过粘贴文本、上传 PDF/Word、上传图片三种方式导入题目。系统先通过正则表达式高速解析并计算置信度，对高置信度（≥0.8）结果直接采纳，中置信度（0.5~0.8）结果调用 AI 逐题修复，低置信度（<0.5）结果交由 AI 整段重新解析。解析完成后进行 MD5 语义指纹去重和质量检查，合格题目批量存入数据库，最终生成导入报告。

**流程图**：

```
用户输入（文本/文件/图片）
       ↓
文本清洗/文件提取/图片OCR
       ↓
正则解析（带置信度）
       ↓
≥0.8 直接采纳 │ 0.5~0.8 AI修复 │ <0.5 AI重解析
       ↓
MD5语义去重
       ↓
质量检查（空题干/缺答案/异常选项）
       ↓
自动修复 / 标记跳过
       ↓
批量入库 → 生成导入报告
```

#### 模块名称：刷题练习模块

**描述**：用户选择练习模式（随机刷题、错题重练、难度专项）、学科、题型和题量后，系统从题库中按条件抽取题目。用户作答后，系统通过多层匹配逻辑判定正误——单选/多选题采用排序后精确匹配，判断题进行选项映射（"对"↔"A"），填空题和简答题依次尝试精确匹配、去括号匹配、包含匹配、多答案子集匹配和 Levenshtein 模糊匹配。每次答题后自动记录答题时间、对错结果，并将练习进度持久化到数据库；用户关闭页面后重新进入可恢复未完成的练习。支持基于艾宾浩斯遗忘曲线（第 1、2、4、7、15、30 天）的间隔复习模式，系统根据答题历史自动推送需要复习的题目。

**流程图**：

```
用户选择模式（随机/错题/难度）+ 参数（学科/题型/数量）
       ↓
检查是否有未完成练习 ──是──→ 恢复上次进度
       ↓ 否
从题库抽题 → 初始化练习会话
       ↓
┌─ 循环 ─────────────────────────────┐
│  展示题目                           │
│       ↓                             │
│  用户作答                           │
│       ↓                             │
│  check_answer() 多层匹配判断正误    │
│       ↓                             │
│  显示正确答案 + 解析                │
│       ↓                             │
│  保存答题记录 + 持久化练习进度      │
└─────────────────────────────────────┘
       ↓
全部答完 → 生成练习报告（正确率/用时/各题型统计）
```

#### 模块名称：模拟考试模块

**描述**：用户设置考试参数（题数、时长、题型范围、学科范围），系统按参数从题库中随机抽题生成试卷。考试界面实时显示倒计时和答题进度条，当剩余时间不足 5 分钟时发出视觉警告；时间用尽后系统锁定答题界面并自动交卷，同时通过 JavaScript 定时器触发页面刷新确保强制生效。交卷后系统逐题评分，展示成绩报告，包括总分、正确率、总用时，以及每道题的用户答案、正确答案对比和详细解析，帮助用户复盘考试表现。

**流程图**：

```
用户设置考试参数（题数/时长/题型/学科）
       ↓
随机抽题生成试卷
       ↓
进入考试界面（倒计时 + 进度条）
       ↓
┌─ 循环 ─────────────────────────────┐
│  用户答题                           │
│       ↓                             │
│  实时检查剩余时间                   │
│  ├─ < 5 分钟 → 警告提示             │
│  └─ ≤ 0 → 强制交卷                 │
│       ↓                             │
│  切换到下一题                       │
└─────────────────────────────────────┘
       ↓
交卷（手动/自动）
       ↓
逐题评分 → 生成成绩报告
（总分/正确率/用时/逐题详情）
```

#### 模块名称：学习统计模块

**描述**：读取当前用户的答题记录，从时间维度、题型维度、正确率维度进行多角度统计分析。使用 Plotly 生成交互式图表：近 30 天核心指标卡片（答题总次数、正确数、正确率）展示学习概况；近 7 天每日答题数量柱状图展示学习强度变化；近 7 天每日正确率趋势折线图展示能力提升轨迹；各题型正确率柱状图对比不同题型的掌握程度。系统根据统计数据自动生成智能学习建议，例如"多选题正确率仅 45%，建议加强多选题专项练习"或"近 3 天未答题，建议保持每日练习节奏"，引导用户针对性复习。

**流程图**：

```
读取当前用户答题记录（practice_records）
       ↓
┌─ 多维度统计 ──────────────────────────┐
│  · 近 30 天：总次数 / 正确数 / 正确率  │
│  · 近 7 天：每日答题量（柱状图）       │
│  · 近 7 天：每日正确率（折线图）       │
│  · 全量：各题型正确率（柱状图）        │
└────────────────────────────────────────┘
       ↓
薄弱环节分析
（正确率 < 60% 的题型标记为薄弱）
       ↓
生成智能建议文本
       ↓
Plotly 交互式图表 + 建议卡片 → 渲染展示
```

#### 模块名称：用户认证模块

**描述**：用户注册时输入用户名、密码和显示名称，系统生成 32 字符随机盐，将盐与密码拼接后进行 SHA-256 哈希，以"盐$哈希"格式存入数据库，确保密码明文不落盘。登录时从数据库取出存储值，分离盐和哈希值，对用户输入的密码执行相同的加盐哈希操作后比对。验证通过后将用户 ID、用户名、显示名称、角色写入 `session_state` 维护会话状态。系统通过 `user_id` 字段实现数据隔离，所有查询（答题记录、收藏、练习会话、导入历史）均附加用户过滤条件，不同用户的数据完全独立、互不可见。

**流程图**：

```
用户访问系统
       ↓
检查 session_state.user_id
       ↓
┌─ 未登录 ──────────────────────────┐
│  显示登录/注册页面                 │
│  ├─ 注册：                        │
│  │    用户名查重                   │
│  │    生成随机盐(32字符)           │
│  │    SHA-256(盐+密码)             │
│  │    存储 盐$哈希 → users 表      │
│  │         ↓                       │
│  │    提示"注册成功，请登录"       │
│  └─ 登录：                        │
│       查询 users 表                │
│       分离 盐 和 哈希值            │
│       SHA-256(盐+输入密码)         │
│       比对哈希值                   │
│       ├─ 匹配 → 写入 session_state │
│       └─ 不匹配 → 提示"密码错误"  │
└────────────────────────────────────┘
       ↓
已登录 → 进入系统主界面
（所有数据查询附加 user_id 过滤）
```
